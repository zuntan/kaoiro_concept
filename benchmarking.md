# 競合比較と独自性の分析 (benchmarking.md)

## 1. 類似アプリケーション・技術の調査結果

調査日: 2026-02-05

| 名称 | カテゴリ | 主な特徴 | kaoiroとの決定的な違い |
| :--- | :--- | :--- | :--- |
| **Project Gameface** | ゲーマー向けアクセシビリティ | 52種類の表情でマウス/キー操作。Google製。 | マウス操作が主であり、開発作業を阻害する可能性がある。多機能すぎて設定が複雑。 |
| **Smyle Mouse** | 身体障害者向け支援 | 頭の動きでマウスを制御。高精度だが有償。 | 「マウスの代替」が目的。日常的なコミュニケーションツールとしては重厚すぎる。 |
| **Nodding Pigeon** | 開発者向けライブラリ | Python/MediaPipeによる頷き検知API。 | 単なるライブラリであり、アプリとして常駐・利用できる形になっていない。 |
| **VSeeFace** | VTuber向けツール | 高精度な表情トラッキング。 | キャラクターを動かすためのものであり、PC操作へのフィードバック機能はない。 |

## 2. kaoiro の独自性 (Unique Selling Points)

### 2.1. 日常に溶け込むカジュアルさ
「アクセシビリティツール」や「開発専用ツール」という枠を超え、誰でもチャットやSNSで「スタンプを押す感覚」で使える親しみやすさを持つ。

### 2.2. 「非侵襲的」な操作感
マウスカーソルを動かさないため、既存の作業フローを全く邪魔しない。必要な時だけ「頷く」ことで、手がキーボードのホームポジションにある状態を維持できる。

### 2.3. AI Native (MCP対応)
単なるキーボードエミュレータに留まらず、将来的に **Model Context Protocol (MCP)** に対応することで、AIエージェント側が「ユーザーがいま頷いた」というイベントを直接受信できる設計となっている。

### 2.4. プライバシーへの徹底した配慮
ユーザーの自宅環境が映り込むことを防ぐため、標準で「顔以外のマスク・ぼかし」機能を搭載する。

## 3. 市場におけるポジショニング

kaoiro は、以下の図の **「Casual & Friendly / AI-Native」** 領域における唯一のツールを目指す。

```text
[ 操作の複雑さ ]
   ^
   |  (Gameface, Smyle Mouse)
   |  アクセシビリティ・全操作代替
   |
   +-------------------------------------> [ AI親和性・カジュアル ]
   |
   |              (kaoiro)
   |              YES/NO特化・コミュニケーション
   v
```

## 4. 結論
技術的な競合は存在するが、**「AIとのカジュアルなコミュニケーション」** という文脈でパッケージングされた製品は皆無である。この新しい市場において、kaoiro はデファクトスタンダードを目指せる。
